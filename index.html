API X SPA

O que é? Propósito Pontos Positivos Pontos Negativos

O QUE É?

API

Ao pé da letra API significa Application Programing Interface ou Interface de Programação de Aplicação, em português. Mas o que isso quer dizer? Esta sigla faz menção ao processo que há por de trás da maioria dos aplicativos, geralmente, realizando a comunicação entre o front-end(parte visual) e back-end(parte operacional), tratamento de dados e validações.

SINGLE APPLICATION

Aplicações de páginas únicas, ou single applications é uma outra forma de criar, implementar e carregar um site. Em contrapartida ao uso de API’s, as single applications visam utilizar menos o back-end/server side e alocar maior conteúdo no lado do usuário, assim diminuindo o tempo de resposta já que as requisições serão mínimas.

PROPÓSITO

API

No processo de criação de uma API os desenvolvedores focam principalmente na abstração e facilidade de implementação a diferentes aplicações, ao mesmo passo ela cria uma padronização que garante maior segurança e coesão às regras de negócio.

SINGLE APPLICATION

Um dos principais objetivos de desenvolver um site em single application é proporcionar uma experiência de usuário mais rápida, interativa e eficiente, que se assemelhe a aplicativos nativos e reduzir a sobrecarga do servidor.

PONTOS POSITIVOS

API

  • ACELERAÇÃO DO DESENVOLVIMENTO: com a implementação de API’s públicas já existentes podemos pular etapas de desenvolvimento pois serão mantidas por terceiros.
  • INTEGRAÇÃO: permite maior integração entre aplicativos e recursos back-end, como banco de dados e outros scripts.
  • AUTOMATIZAÇÃO DE TAREFAS: As APIs podem ser usadas para automatizar tarefas repetitivas, economizando tempo e minimizando erros humanos.

SINGLE APPLICATION

  • EXPERIÊNCIA INTERATIVA: Oferece uma experiência de usuário mais interativa e responsiva, pois permitem a atualização do conteúdo sem recarregar toda a página.
  • REDUÇÃO DE REQUISIÇÕES DE SERVIDOR: Minimizam as requisições de servidor, buscando dados apenas quando necessário.
  • MANUTENÇÃO SIMPLES: Devido à separação clara entre o front-end e o back-end, as elas podem ser mais fáceis de manter e atualizar.

PONTOS NEGATIVOS

API

  • NECESSIDADE DE AUTENTICAÇÃO: Devido a comunicação com meios externos a validação de requisições e respostas do sistema se torna essencial para a segurança de todo ecossistema, já que este é um ponto de vulnerabilidade e ataques externos.
  • COMPLEXIDADE DE CÓDIGO: A lógica e linguagem de programação back-end são de complexa compreensão, por isso há uma dificuldade de implementação para programadores menos experientes, criando uma maior dependência da documentação da API.
  • CUSTOS DE MANUTENÇÃO: Manter uma API requer recursos contínuos, incluindo suporte técnico, atualizações, correções de segurança e escalabilidade. Isso pode gerar custos significativos ao longo do tempo.

SINGLE APPLICATION

  • MECANISMOS DE PESQUISA DIFICULTADOS: As single applications tinham desafios de busca devido à falta de conteúdo indexável nas páginas.
  • DESENVOLVIMENTO COMPLEXO: O desenvolvimento de uma single application pode ser mais complexo do que o de sites tradicionais devido ao gerenciamento de estados no lado do cliente e implementação de estratégias para carregamento eficiente.
  • COMPATIBILIDADE: Nem todos os navegadores suportam recursos avançados de JavaScript usados em single applications, o que pode exigir o uso de polyfills (códigos extras para preencher lacunas de compatibilidade) ou limitar a compatibilidade com versões mais antigas dos navegadores.
